# 🚀 Imperva WAF AIOps 智能助手 (基于 MCP 架构)

## 📖 项目概述
本项目旨在探索大语言模型（LLM）与企业级安全防护基础设施的深度融合。通过引入标准的 **MCP (Model Context Protocol)** 架构，我们成功为云端大模型赋予了直连本地 Imperva SecureSphere WAF 的能力。

这意味着，安全运营人员不再需要频繁登录 WAF 图形界面进行繁琐的点击和查询，而是可以**直接通过自然语言对话，让 AI 自动完成安全告警巡检、黑名单排查等高频运维工作**，真正迈入 AIOps（智能运维）时代。

## ✨ 核心核心价值 (Value Proposition)
* **🗣️ 交互降维：** 将复杂的 API 调用和 GUI 操作转化为简单的自然语言对话。
* **⚡ 效率倍增：** 瞬间拉取并解析海量 JSON 告警数据，AI 自动提炼核心威胁指标（如高频攻击源、受威胁站点），秒出巡检报告。
* **🔒 安全可控：** 采用标准的权限隔离架构，大模型仅负责决策和意图识别，所有真实网络请求均由本地部署的 MCP Server 发起，且完美兼容企业内网 VPN 环境，确保数据不出域。

## 🏗️ 系统架构设计
本项目采用了经典的分层解耦架构，数据流向清晰，具备极强的可扩展性：

1. **MCP Client (大模型客户端)：** 如 Cherry Studio，负责作为人机交互主界面，并调度大模型理解用户意图。
2. **LLM (云端大脑)：** 负责意图识别与数据分析，本身不具备内网访问权限。
3. **MCP Server (本地特工)：** 基于 Node.js 开发的模块化中间件。它接收大模型的工具调用指令，在本地安全环境内（如连接企业 VPN 后）向 WAF 发起 RESTful API 请求。
4. **Imperva WAF (执行终端)：** 接收合法的 API 请求并返回底层安全数据。

## 🛠️ 当前已解锁技能 (Capabilities)
目前该智能助手已成功注册并实装了以下核心安全工具：
* 🛡️ `get_waf_security_events`: **安全告警智能巡检**。支持自定义时间范围（如过去 24 小时），拉取最新安全违规事件，并自动生成结构化的威胁概况报告。
* 🚫 `get_waf_blocked_sources`: **封禁源实时排查**。一键获取当前 WAF 上所有被拉黑的恶意 IP 及其封禁原因。

## 🚀 快速部署与体验指南

### 1. 环境准备
* 确保本机已安装 **Node.js** (推荐 v18 及以上版本)。
* 确保本机已连接企业内网或 **VPN**，能够正常 Ping 通 WAF 管理中心 IP。

### 2. 获取与配置项目
将项目代码克隆/复制到本地目录后，在根目录创建或修改 `.env` 环境配置文件：

```env
# WAF 管理中心的基础地址与端口
WAF_BASE_URL=https://<WAF真实的IP地址>:8083

# 拥有 API 调用权限的管理员账号
WAF_USER=admin

# WAF 密码（⚠️ 极其重要：若密码包含特殊字符，必须使用英文双引号包裹，防止被截断）
WAF_PASSWORD="YourPassword123!*"
```

### 3. 安装依赖与启动
在项目根目录执行以下命令安装运行所需的依赖包：
```bash
npm install axios dotenv @modelcontextprotocol/sdk --legacy-peer-deps
```
*(注：本项目已深度优化网络模块，强制屏蔽了桌面端代理软件的干扰，确保纯净的内网直连体验。)*

### 4. 接入客户端 (以 Cherry Studio 为例)
1. 打开 Cherry Studio 设置 -> **MCP**。
2. 点击新增，类型选择 `stdio`。
3. **命令 (Command):** 填入本机 node 程序的绝对路径 (如 `/usr/local/bin/node`)。
4. **参数 (Arguments):** 填入本项目 `index.js` 的绝对路径 (如 `/Users/xxx/imperva-mcp/index.js`)。
5. 开启服务，当绿灯亮起，且主界面输入框上方出现“可用工具”提示时，即代表接入成功！
